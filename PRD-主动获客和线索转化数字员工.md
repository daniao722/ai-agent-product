# PRD：主动获客和线索转化数字员工

## 1. 摘要

本文档定义了主动获客和线索转化数字员工产品的需求规格。这是一个基于多Agent协作的AI产品，具备两大核心能力：1）通过海关数据等应用从目标国家市场智能查询匹配主营业务和产品的采购商，自动生成开发信EDM或营销内容，客户确认后自动执行发送任务；2）对从各个渠道转化来的客户线索自动打标，并创建EDM发送、社媒推送等营销任务，客户同意之后自动执行。每个数字员工都有独立的工作台看板，可以监测工作状态和效果数据变化。

## 3. 背景

### 3.1 上下文

外贸企业在主动获客和线索转化方面面临以下痛点：
- **获客效率低**：手动查找采购商费时费力，数据分散
- **开发信质量差**：难以写出个性化、高回复率的开发信
- **线索跟进不及时**：线索分配、打标、跟进依赖人工，容易遗漏
- **营销任务繁重**：EDM、社媒等多渠道营销需要大量人力
- **效果难以追踪**：哪个渠道、哪封邮件带来的转化不清楚

### 3.2 为什么是现在？

1. **AI技术成熟**：大语言模型在个性化内容生成、数据理解方面能力突出
2. **数据可用**：海关数据、采购商数据等商业数据逐渐开放和标准化
3. **市场需求迫切**：外贸企业从"等客上门"转向"主动出击"的需求强烈
4. **战略对齐**：符合公司"数智增长合伙人"的战略定位，帮助客户实现增长

- 参加展会：成本高，周期长，效果不确定

## 7. 解决方案

### 7.1 两大核心能力

#### 7.1.1 能力一：智能采购商发掘与开发信自动化

**功能描述：**
通过海关数据等多源数据，智能查询匹配主营业务和产品的采购商，自动生成个性化开发信EDM，客户确认后自动执行发送任务。

**数据来源：**
1. **海关数据**：目标国家的进出口记录
2. **采购商数据库**：商业数据提供商
3. **搜索引擎**：Google、LinkedIn等
4. **社媒数据**：LinkedIn、Facebook等
5. **企业官网**：采购商官网信息爬取

**核心流程：**

```
用户设置：目标市场 + 产品关键词 + 筛选条件
    ↓
采购商发掘Agent搜索多源数据
    ↓
AI智能匹配和评分
    ↓
生成采购商列表（带联系方式）
    ↓
用户确认/调整采购商名单
    ↓
开发信生成Agent为每个采购商写个性化开发信
    ↓
用户预览/编辑开发信
    ↓
确认后自动发送EDM
    ↓
追踪邮件打开、点击、回复
    ↓
自动触发后续跟进任务
```

#### 7.1.2 能力二：全渠道线索管理与自动化营销

**功能描述：**
对从各个渠道转化来的客户线索自动打标，并创建EDM发送、社媒推送等营销任务，客户同意之后自动执行。

**线索接入渠道：**
1. **网站询盘**：官网、落地页
2. **社媒私信**：LinkedIn、Facebook、Instagram
3. **邮件咨询**：直接邮件、开发信回复
4. **展会收集**：名片扫描、表单录入
5. **第三方线索**：B2B平台、数据购买

**核心流程：**

```
线索从各渠道接入
    ↓
线索清洗和去重
    ↓
线索打标Agent自动打标
    ├─ 客户类型（潜在/意向/热）
    ├─ 行业标签
    ├─ 来源渠道
    ├─ 意向产品
    └─ 优先级评分
    ↓
创建自动化营销任务
    ├─ EDM nurture序列
    ├─ 社媒内容推送
    ├─ 销售跟进提醒
    └─ 个性化内容推荐
    ↓
用户确认/调整营销计划
    ↓
自动执行营销任务
    ↓
追踪转化效果
    ↓
持续优化策略
```

### 7.3 核心功能详解

#### 7.3.1 智能采购商发掘

**功能描述：**
多源数据搜索，AI智能匹配，生成高意向采购商列表。

**搜索维度：**
1. **产品关键词**：目标产品相关关键词
2. **目标市场**：国家、地区筛选
3. **采购金额**：按采购规模筛选
4. **行业类别**：HS编码、行业分类
5. **企业规模**：员工数、营收规模
6. **近期活跃度**：最近采购记录

**输出：**
- 采购商列表（按意向评分排序）
- 企业信息：公司名、地址、官网、社媒
- 联系人信息：姓名、职位、邮箱、电话（如有）
- 采购历史：相关产品采购记录
- 匹配评分：AI计算的匹配度（0-100分）

2. **采购历史**：相关产品采购记录
3. **社媒动态**：LinkedIn、官网最新动态
4. **产品匹配**：我们的产品如何解决他们的问题
5. **时机把握**：最近是否有采购行为

**开发信类型：**
1. **首封开发信**：破冰、自我介绍、价值主张
2. **跟进邮件**：基于首封回复情况的个性化跟进

- 多版本开发信（A/B测试用）

- 发送时间建议（最佳打开时间）

#### 7.3.3 全渠道线索接入与自动打标

**功能描述：**
统一接入各渠道线索，AI自动打标、评分、分类。

**接入渠道：**
1. **官网表单**：Webhook、API接入
2. **社媒平台**：LinkedIn、Facebook、Instagram API
3. **邮件系统**：Gmail、Outlook、企业邮箱
4. **B2B平台**：Alibaba、Made-in-China等
5. **手动录入**：Excel导入、名片扫描
6. **展会数据**：展会收集的线索批量导入

**打标维度：**
1. **线索热度**：
   - 🔴 热线索：有明确购买意向、预算、 timeline
   - 🟡 温线索：有兴趣、需要培养
   - 🟢 冷线索：初步接触、需要长期培育

2. **客户类型**：
   - 终端客户
   - 经销商/代理商
   - 系统集成商
   - 其他

3. **行业标签**：
   - 制造业、医疗、零售等
   - 基于公司信息自动识别

4. **来源渠道**：
   - 官网、LinkedIn、邮件、展会等
   - 记录首次接触渠道

5. **意向产品**：
   - 基于询盘内容自动识别
   - 匹配我们的产品线

6. **优先级评分**：
   - 0-100分综合评分
   - 考虑：预算、决策权、急迫性、匹配度

**输出：**
- 标准化线索档案
- 多维度标签体系
- 线索评分和排序
- 自动分配给销售
- 后续行动建议

#### 7.3.4 自动化营销任务创建与执行

**功能描述：**
基于线索标签和客户旅程，自动创建EDM发送、社媒推送等营销任务，客户同意后自动执行。

**营销任务类型：**
1. **EDM Nurture序列**：
   - 欢迎邮件（新线索）
   - 案例分享（培养中）
   - 产品推荐（有兴趣）
   - 限时优惠（接近决策）
   - 唤醒邮件（沉默线索）

2. **社媒内容推送**：
   - LinkedIn Connection请求
   - 个性化社媒消息
   - 内容点赞、评论
   - 公司内容@提及

3. **销售跟进提醒**：
   - 电话跟进提醒
   - 见面日程安排
   - 报价发送提醒
   - 合同跟进提醒

4. **个性化内容推荐**：
   - 基于行业的案例
   - 基于产品的白皮书
   - 基于角色的内容

**客户旅程自动化：**

```
新线索接入
    ↓
[热线索] ──────→ 销售直接跟进 + 高价值内容
    ↓
[温线索] ──────→ Nurture序列（3-5封邮件）+ 社媒互动
    ↓
[冷线索] ──────→ 长期培育（每月1-2次）+ 品牌内容
    ↓
线索活跃（打开/点击/回复）→ 升级线索热度 → 调整营销策略
    ↓
线索沉默（30天无互动）→ 唤醒邮件 → 仍无反应则降低优先级
```

- 自动化营销任务计划
- 任务执行日历
- A/B测试方案
- 效果追踪设置
- 优化建议

### 7.7 假设与风险

**假设：**
1. 采购商数据的质量和可获得性满足要求
2. AI生成的开发信回复率可以达到10%+
3. 外贸企业愿意让AI自动化执行营销任务（经过确认）
4. 多渠道线索统一接入在技术上可行

**风险：**
1. 数据合规和隐私问题 → 严格遵守GDPR等法规，用户授权
2. AI生成内容质量不稳定 → 提供人工审核和编辑
3. 邮件送达率和回复率低于预期 → 持续优化模板和策略
4. 社媒平台API限制 → 多种数据来源备份
